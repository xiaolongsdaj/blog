# 博客后端 API 接口文档

## 1. 认证相关接口

### 1.1 用户注册

- **请求方法**: POST
- **请求路径**: `/api/auth/register`
- **认证要求**: 不需要
- **请求体**:
  ```json
  {
    "username": "string", // 用户名，必填
    "email": "string", // 邮箱，必填
    "confirmPassword": "string", // 确认密码，必填
    "password": "string" // 密码，必填
  }
  ```
- **返回值**:
  ```json
  {
    "success": true,
    "message": "用户注册成功",
    "data": {
      "user": {
        "id": "number",
        "username": "string",
        "email": "string",
        "role": "string",
        "createdAt": "string",
        "isActive": "boolean"
      },
      "accessToken": "string",
      "refreshToken": "string"
    }
  }
  ```

### 1.2 用户登录

- **请求方法**: POST
- **请求路径**: `/api/auth/login`
- **认证要求**: 不需要
- **请求体**:
  ```json
  {
    "emailOrUsername": "string", // 邮箱或用户名，必填
    "password": "string" // 密码，必填
  }
  ```
- **返回值**:
  ```json
  {
    "success": true,
    "message": "登录成功",
    "data": {
      "user": {
        "id": "number",
        "username": "string",
        "email": "string",
        "role": "string",
        "avatar": "string",
        "bio": "string",
        "createdAt": "string",
        "isActive": "boolean"
      },
      "accessToken": "string",
      "refreshToken": "string"
    }
  }
  ```

### 1.3 刷新令牌

- **请求方法**: POST
- **请求路径**: `/api/auth/refresh-token`
- **认证要求**: 不需要
- **请求体**:
  ```json
  {
    "refreshToken": "string" // 刷新令牌，必填
  }
  ```
- **返回值**:
  ```json
  {
    "success": true,
    "message": "令牌刷新成功",
    "data": {
      "accessToken": "string"
    }
  }
  ```

### 1.4 注销登录

- **请求方法**: POST
- **请求路径**: `/api/auth/logout`
- **认证要求**: 需要
- **返回值**:
  ```json
  {
    "success": true,
    "message": "注销成功"
  }
  ```

## 4. 认证说明

### 4.1 认证方式

- 使用 JWT（JSON Web Token）进行认证
- 登录成功后会返回`accessToken`和`refreshToken`
- `accessToken`有效期较短，用于 API 请求认证
- `refreshToken`有效期较长，用于刷新`accessToken`

### 4.2 请求认证

- 在需要认证的 API 请求头中添加`Authorization`字段
- 格式：`Bearer <accessToken>`

### 4.3 认证失败响应

```json
{
  "success": false,
  "message": "认证失败",
  "error": "具体错误信息"
}
```

## 5. 错误码说明

| 错误码 | 说明             |
| ------ | ---------------- |
| 400    | 请求参数错误     |
| 401    | 未认证或认证失败 |
| 403    | 权限不足         |
| 404    | 资源不存在       |
| 500    | 服务器内部错误   |

## 6. 示例请求

### 使用 curl 请求登录接口

```bash
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"emailOrUsername": "test@example.com", "password": "password123"}'
```

### 使用 axios 请求获取文章列表

```javascript
import axios from "axios";

const token = "your_access_token";

axios
  .get("http://localhost:3000/api/articles", {
    headers: {
      Authorization: `Bearer ${token}`,
    },
    params: {
      page: 1,
      limit: 10,
      categoryId: 1,
    },
  })
  .then((response) => {
    console.log(response.data);
  })
  .catch((error) => {
    console.error(error);
  });
```

## 8. 注意事项

1. 所有请求和响应数据格式均为 JSON
2. 日期时间格式为 ISO 8601 格式
3. 分页接口返回数据包含总条数、页码、每页数量和总页数
4. 认证接口返回的 token 需要妥善保存，避免泄露
5. 生产环境中需要配置 HTTPS，确保数据传输安全
